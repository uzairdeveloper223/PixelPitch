# Security Policy

## Supported Versions

Currently, only the latest version of PixelPitch on the `main` branch is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

Security is a high priority for this project. If you discover a vulnerability in PixelPitch, **please do not report it in a public GitHub issue**. Instead, report it privately so it can be verified and patched securely before public disclosure.

**Please contact us by email:**
`contact@uzair.is-a.dev`

### What to include in your report:

- A clear, detailed description of the vulnerability.
- Step-by-step instructions or a proof of concept (PoC) to reproduce the issue.
- The environment details where the issue was discovered (Browser, OS version, etc.).
- Any thoughts on potential fixes or mitigations.

### When to expect a response:

You should receive an acknowledgment within 48-72 hours. From there, we will provide updates on the triage process, coordinate a patch timeline, and discuss public disclosure under responsible disclosure guidelines.
